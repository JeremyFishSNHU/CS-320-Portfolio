Reflection
How can I ensure that my code, program, or software is functional and secure?

I can ensure that my code is functional by carefully reviewing the requirements, writing code that directly supports those requirements, and creating unit tests to verify that each part of the program works correctly. For example, in the contact service project, I used JUnit tests to check that contacts could be created, updated, and deleted properly. Testing also helped me catch possible errors before the program was considered complete.

To help make software more secure, I can validate user input, avoid allowing invalid data, and make sure the program handles errors correctly. Writing tests for both valid and invalid inputs is important because it helps confirm that the program does not accept data that could cause problems later. Secure software starts with careful design, clear requirements, and consistent testing.

How do I interpret user needs and incorporate them into a program?

I interpret user needs by first reviewing the project requirements and identifying what the program is expected to do. Instead of guessing, I break the requirements into smaller pieces and turn them into specific features. For example, if a requirement says that a contact ID cannot be longer than a certain number of characters or cannot be changed after creation, I make sure that rule is included in the code and tested.

Incorporating user needs into a program means making sure the final product solves the problem the user actually has. Testing helps with this because each test can be connected back to a requirement. This allows me to confirm that the program is not only working, but also meeting the expectations of the user.

How do I approach designing software?

When I design software, I start by understanding the requirements and planning the structure of the program before writing code. I think about what classes, methods, and services are needed, and I try to keep each part focused on one responsibility. For the contact service project, the Contact class handled the contact information, while the ContactService class handled actions such as adding, updating, and deleting contacts.

I also approach software design with testing in mind. If code is organized clearly, it is easier to test and maintain. Good design helps prevent confusion, makes future updates easier, and reduces the chance of errors.
